# BYAGENT - AI Trading Signals Platform

## Деплой на Render

### Шаг 1: Подготовка репозитория

1. Создайте новый репозиторий на GitHub
2. Загрузите все файлы из этого архива в репозиторий

### Шаг 2: Создание Web Service на Render

1. Зайдите на [render.com](https://render.com) и войдите в аккаунт
2. Нажмите **"New +"** → **"Web Service"**
3. Подключите ваш GitHub репозиторий
4. Настройте параметры:
   - **Name**: `byagent` (или любое другое имя)
   - **Region**: выберите ближайший регион
   - **Branch**: `main` (или ваша основная ветка)
   - **Runtime**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: `Free` (или платный план для лучшей производительности)

### Шаг 3: Настройка переменных окружения

В разделе **Environment Variables** добавьте:

#### Обязательные переменные:

- `PORT` = `10000` (автоматически устанавливается Render)

#### Опциональные переменные (для полного функционала):

- `TELEGRAM_BOT_TOKEN` = ваш токен Telegram бота (получите у @BotFather)
- `TELEGRAM_CHAT_ID` = ID чата для отправки сигналов
- `MONGODB_URI` = строка подключения к MongoDB (можно использовать MongoDB Atlas)
- `COINGECKO_API_KEY` = API ключ CoinGecko (опционально, для увеличения лимитов)

### Шаг 4: Деплой

1. Нажмите **"Create Web Service"**
2. Render автоматически начнет деплой
3. Дождитесь завершения (обычно 2-5 минут)
4. Ваше приложение будет доступно по адресу: `https://your-app-name.onrender.com`

### Шаг 5: Настройка MongoDB (опционально)

Если вы хотите использовать базу данных:

1. Создайте бесплатный кластер на [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. Получите строку подключения
3. Добавьте IP адрес `0.0.0.0/0` в whitelist (для Render)
4. Добавьте строку подключения в переменную окружения `MONGODB_URI`

### Шаг 6: Настройка Telegram бота (опционально)

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен бота
3. Узнайте ID вашего чата (можно через [@userinfobot](https://t.me/userinfobot))
4. Добавьте переменные `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`

## API Endpoints

- `GET /` - Главная страница (index.html)
- `GET /api/signals` - Получить текущие торговые сигналы
- `GET /api/health` - Проверка статуса сервера
- `POST /api/webhook` - Webhook для получения сигналов извне

## Особенности

- ✅ Полностью рабочий интерфейс без изменений
- ✅ Та же логика генерации сигналов
- ✅ Автоматическая отправка GOD TIER сигналов в Telegram каждые 2 минуты
- ✅ Поддержка MongoDB для хранения истории сигналов
- ✅ WebSocket поддержка
- ✅ CORS включен для API запросов

## Важные замечания

1. **Free tier на Render**: бесплатный план засыпает после 15 минут неактивности. Первый запрос после сна может занять 30-60 секунд.

2. **Cron задачи**: работают только когда сервер активен. Для постоянной работы рекомендуется платный план.

3. **MongoDB**: не обязателен для базовой работы, но рекомендуется для хранения истории сигналов.

4. **CoinGecko API**: бесплатный план имеет лимиты. Для production рекомендуется получить API ключ.
